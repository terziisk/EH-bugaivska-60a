Прочитай `@CURRENT_PHASE.md` и убедись, что фаза = `phase_3`.

Прочитай:
- `@01_research/phase1_niches/niches_master.md` — финальный шорт-лист
  компаний из Фазы 2 (~50 шт).
- `@00_object/object_specification.md` — для контекста объекта.
- `@CLAUDE.md` — стиль работы.

## Шаг 1. Подготовка плана

Из `niches_master.md` извлеки полный список компаний из секции
«Сводный шорт-лист компаний». Это вход для Фазы 3.

Если в шорт-листе > 60 компаний — оставь топ-50 по полю **Фит (1–5)**,
чтобы не перегружать ресерч. Уточни этот список со мной перед
запуском, если порезка требуется.

## Шаг 2. Параллельный запуск company-researcher

Для **КАЖДОЙ** компании из списка спавни параллельно subagent
`company-researcher` через Task tool. В промпт каждого передай:
- название компании (UA + EN если оба известны)
- ниша и сценарий из niches_master
- 2–3 строки контекста: почему мы её выбрали (поле «Активность
  расширения 2025–2026» из шорт-листа)
- путь к `00_object/object_specification.md`
- напоминание: главный приоритет в контактах — **email** (общий
  отдела развития / прямой / форма «Запропонувати локацію»).

Дождись завершения всех агентов.

**Важно:** при больших объёмах (50+ агентов параллельно) могут быть
лимиты — запускай батчами по 10–15 агентов, ждать завершения,
запускать следующий батч.

## Шаг 3. Сводный отчёт

Прочитай все файлы из `01_research/phase2_companies/by_company/`.
Создай `01_research/phase2_companies/companies_master.md`:

```markdown
# Сводный отчёт по компаниям (Фаза 3)

Дата: <дата>
Всего исследовано: <N> компаний
С найденным email: <M>
С формой «Suggest location»: <K>

## Сводная таблица

| Компания | Ниша | Сценарий | Фит-скор (1–10) | Контакт найден? | Канал #1 | Источник |
|---|---|---|---|---|---|---|

## Особые находки

- Компании с активной экспансией в Одессу 2026: ...
- Компании с публичной формой «Suggest location»: ...
- Компании, где не нашли никаких контактов: ...
```

## Шаг 4. Обновить трекинг

- Обновить `LOG.md`: что сделано, размер выборки, сколько компаний с
  найденными email, какие неожиданные находки.
- Обновить `CURRENT_PHASE.md` → `phase_4`.

## Что НЕ делать

- Не дублировать ресерч из Фазы 2 (та работала по нишам, эта — по
  отдельным компаниям).
- Не обогащать профили вручную, если company-researcher что-то не
  нашёл — фиксируй «Не підтверджено» и иди дальше.
- Не записывать ничего в `02_priority/` — это работа Фазы 4
  (`prioritize`).
