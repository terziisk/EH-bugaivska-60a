## Шаг 1. Скоринг каждой компании

Для каждой компании из `companies_master.md` посчитай **итоговый
скор приоритизации** по 4 факторам, каждый 1–5 (итого max 20):

1. **Фит с объектом** — соответствие нашего формата (1500 м² standalone
   + двор + параметры) и их типового запроса по аренде.
2. **Активность расширения 2025–2026** — открывают ли новые точки прямо
   сейчас, есть ли публичные планы, заявлена ли Одесса.
3. **Достижимость** — есть ли email отдела развития, форма «Suggest
   location», LinkedIn ответственного. Без контакта = 1.
4. **Стратегическое соответствие** — насколько наш объект решает
   их конкретную задачу (а не просто «у них есть запрос на аренду»):
   например, формат «standalone-флагман с парковкой и фасадом» —
   плюс для DIY/мебели; «отдельное здание с двором» — плюс для
   офис-релокации; «электромощность 50 кВт без газа» — плюс для
   IT-производства, минус для пищевого с газовыми печами.

Сложи 4 балла → итоговый скор 4–20.

## Шаг 2. Деление на Tier

- **Tier 1 (top 8–10):** скор ≥ 16, найден контакт, есть активность
  расширения. Это первоочередные — для них пойдёт outreach в Фазе 5.
- **Tier 2 (15–20):** скор 12–15 или скор ≥ 16 без контакта (нужно
  доискать).
- **Tier 3 (остальные):** скор < 12 или явные блокеры (политика
  «только ТРЦ», требование высоты потолков 6 м, газовое производство).

**Минимум Tier 1 = 8 компаний. Максимум = 12.** Если по скорингу
получается < 8 — снизь порог на 1 балл и добавь.

## Шаг 3. Структура папок

Создай папки:
- `02_priority/tier1/`
- `02_priority/tier2/`
- `02_priority/tier3/`

Внутри каждой — папка по компании по slug-имени (lowercase, латиницей,
без пробелов): `02_priority/tier1/sinsay/`, `tier1/jysk/` и т. п.

В каждую папку компании **скопируй** профиль из
`01_research/phase2_companies/by_company/<slug>.md` →
`02_priority/tier<N>/<slug>/profile.md`. Это нужно, чтобы пакет
outreach был самодостаточным.
