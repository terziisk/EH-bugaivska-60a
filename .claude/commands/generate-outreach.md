## Аргументы команды

`$ARGUMENTS` — необязательный, значения: `tier1` · `tier2` · `tier3`.
Если пусто — по умолчанию `tier1` (Фаза 5). Для `tier2`/`tier3`
команда работает идентично, только меняется источник списка компаний
и папка назначения.

Примеры запуска:
- `/generate-outreach` — генерация Tier 1 (12 компаний)
- `/generate-outreach tier2` — Tier 2 (20 компаний)
- `/generate-outreach tier3` — Tier 3 (long-shot пул, по запросу)

Прочитай:
- `@02_priority/priority_matrix.md` — список компаний выбранного tier
  и их hooks.
- `@00_object/object_specification.md` — параметры объекта.
- `@CLAUDE.md` — стиль работы.
- `@.claude/agents/letter-writer.md` — правила формата (особенно
  раздел «Канал для відправки — тільки verified email» и
  «Що ЗАБОРОНЕНО»).

## Контекст модели outreach (с 2026-05-08)

Универсальный PDF «Фасад + склад на Дальницкой 1500кв.м..pdf» (19 стр.,
8 концепт-визуализаций под разные форматы бизнеса) — **основное
маркетинговое collateral**, прикладывается ко всем письмам Tier 1/2/3.

**Что нужно сделать в этом запуске:**

1. Определить список компаний выбранного tier из `priority_matrix.md`.
2. **Шаг 0 — валидация контактов** (новый, см. ниже): для каждой
   компании проверить, есть ли в `profile.md` verified email. Без
   verified email лист не отправляется — компания идёт в hold-list.
3. Сгенерировать персонализированные письма UA для тех компаний, у
   которых есть verified email.
4. (Только при первом запуске для Tier 1) Создать одноразовый бриф
   `03_outreach/missing_renders_brief.md`. Для Tier 2/3 этот шаг
   пропускается — бриф уже есть.
5. Обновить (или создать для tier2/tier3) сводный
   `03_outreach/outreach_plan_<tier>.md` (для Tier 1 файл уже создан и
   называется `outreach_plan.md` — оставляем как есть).

**Чего НЕ делать:**

- НЕ запускать `image-prompt-builder` для всех компаний массово. Он
  остаётся как опциональный инструмент для **топ-hero компаний** —
  если оператор захочет персональный рендер с логотипом. По
  умолчанию — пропускаем.
- НЕ генерировать таблицы характеристик в письмах — они уже в PDF.
- НЕ использовать в письме **запрещённые фразы** (см.
  `letter-writer.md` → «Що ЗАБОРОНЕНО»): «цього тижня», «цими
  днями», обещания rent-free / срока договора / скидок. Канал
  отправки — **только email**, LinkedIn/соцсети не используются.

## Шаг 0. Валидация контактов (БЛОКИРУЮЩИЙ для каждой компании)

Перед спавном letter-writer для каждой компании tier:

1. Прочитай `02_priority/<tier>/<slug>/profile.md` → раздел
   «Контакти» (или «Контактна стратегія»).
2. Найди хотя бы один **verified email**:
   - ✅ Считается verified: адрес из официального сайта компании
     (контакт-страница, страница «Розширення мережі / Develop /
     Suggest a location»), из публичной реєстраційної інформації
     (YouControl, Opendatabot), либо явно подтверждённый через
     ZoomInfo / RocketReach как «verified» / «confirmed».
   - ❌ НЕ считается verified: адреса с приписками
     `(розрахунковий)`, `(computed)`, `(передбачуваний)`,
     `(extrapolated)`, `(guessed)`, `(розрахункова за форматом
     first.last@...)`. Даже если корпоративный формат «угадан
     правильно» — половина таких писем уйдёт в bounce, и адресат
     прочитает остальные с минусом доверия.
3. Сформируй три группы:
   - **READY** — есть хотя бы один verified email → запускаем
     letter-writer.
   - **HOLD** — есть только LinkedIn / соцсети / расчётные email →
     letter-writer **не запускается**, компания фиксируется в
     hold-list с явной задачей оператору: зайти на официальный сайт
     компании (Contacts / Про нас / Розширення) и взяти email
     звідти. Если за 1–2 дня операторного ресерча email не
     находится — компания возвращается ресерчеру для повторного
     прохода по профилю.
   - **BLOCKED** — нет никакого канала (ни email, ни LinkedIn, ни
     формы) → компания исключается из текущей хвилі outreach
     полностью.
4. Сохрани отчёт о валидации в `03_outreach/contact_audit_<tier>.md`
   с тремя секциями (READY / HOLD / BLOCKED). По каждой компании из
   HOLD — указать конкретно, где оператор должен искать email
   (главная страница сайта, /contacts, /about, /develop, /partnership).

## Шаг 1. Список целевых компаний

Из `priority_matrix.md` извлеки список компаний выбранного tier и их
slug-папок (`02_priority/<tier>/<slug>/`). Сверь с фактическими
папками — у каждой должен существовать `profile.md`.

Применяется к ним результат **Шага 0**: работаем только с группой
READY.

## Шаг 2. Параллельная генерация писем (батч 4–6 одновременно)

Для **КАЖДОЙ** компании из группы READY спавни subagent `letter-writer`
через Task tool (foreground, как Phase 3 — никогда не в background).

В промпт subagent'у передай:
- название компании
- путь к профилю: `02_priority/<tier>/<slug>/profile.md`
- путь к матрице: `02_priority/priority_matrix.md` (для hook)
- путь сохранения: `02_priority/<tier>/<slug>/letter_uk.md`
- **явное напоминание правил из `letter-writer.md`:**
  - 150–220 слов, hook из priority_matrix, 3 буллета под их боль
  - **БЕЗ** «цього тижня», «цими днями», «якнайшвидше»
  - **БЕЗ** обещаний rent-free, срока договора, скидок, ремонта за наш счёт
  - канал в чек-листе — **только verified email** из profile.md;
    LinkedIn/соцсети не указываем как Канал #1
  - конкретный verified email Канал #1 — передать в промпт
    (полученный на Шаге 0), чтобы агент гарантированно вписал
    именно его, без угадывания формата.

Дождись завершения батча, проверь:
- letter_uk.md создан и <= 220 слов в основном тексте; если перевалил —
  повтори с явным указанием «сократи до 200 слов»;
- в тексте нет запрещённых фраз (grep по `цього тижня`, `цими
  днями`, `rent-free`, `5–10 років`); если есть — повтори с
  указанием «прибрати фразу X згідно з letter-writer.md»;
- в чек-листе пакета указан именно тот verified email, что был
  выбран на Шаге 0.
